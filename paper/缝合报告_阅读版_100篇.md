# G-Refer 缝合选题 · 泛读版（100 篇 · 大白话）

> **怎么用这份文档**：按四个方向分块，每篇给【链接】+【它在做什么（大白话）】。
> 标 ⭐ 的是我二次筛选后选进"Agent 执行版 Top-20"的、能直接改进 G-Refer 的论文，额外给一句【缝到G-Refer】。
> 没标星的是背景/候选池，帮你建立领域全貌、第一轮筛选用。
>
> **一句话领域地图**：工业推荐是个漏斗——`召回(海选) → 粗排(初筛) → 精排(决赛) → 重排(排顺序)`。G-Refer 的检索器本质就是在一张用户-商品大图上做同样的事：一步步筛出最有用的 top-k 路径和节点，再喂给 LLM 写推荐理由。所以下面四个方向的成熟套路，都能往 G-Refer 上搬。

# 方向一：检索增强（RAG / 检索增强的推荐与解释）
> 大白话：让模型别光靠脑子里背的，先去"查资料"再回答。G-Refer 就是查图里的 CF 证据。

1. **G-Refer**（本篇基线）https://arxiv.org/abs/2502.12586 — 图检索增强 LLM 做可解释推荐；检索 top-k 路径+节点喂 LLM。**这是你的地基。**

2. ⭐ **K-RagRec** https://arxiv.org/abs/2501.02226 — 把每个商品的知识子图按"多跳"存进向量库，按流行度决定要不要查，查到的子图先重排、再用 GNN+投影层塞进 LLM。**缝到G-Refer**：这就是你想做的"子图级检索"的现成范式，还教你怎么把子图当 soft-prompt 注入而非写成长文本。

3. ⭐ **REXHA（分层聚合检索增强解释）** https://arxiv.org/abs/2507.09188 — 用分层摘要拼用户画像（治"画像跑偏"），用向量检索替掉 O(N²) 最短路（4 分钟→<1 秒）。**缝到G-Refer**：直击你两大痛点——画像偏差 + 检索慢，报告 BERTScore-Precision 高 7.46%–26.8%。

4. **MMP-Refer** https://arxiv.org/abs/2604.03666 — G-Refer 的多模态版：RQ-VAE 融图文 + 规则启发式路径检索替可学边权（快 3–6 倍），BERTScore 高 0.56%–4.01%。（你笔记里已记）

5. **MATRAG（多智能体透明 RAG）** https://arxiv.org/abs/2604.20848 — 多个 agent 分工做检索+生成，强调解释的"透明可追溯"。

6. **MoE-KG-RAG 多智能体推荐** https://arxiv.org/abs/2605.28175 — 用专家混合(MoE)路由不同知识图谱检索，多 agent 协作推荐。

7. **KERAG_R** https://arxiv.org/abs/2507.05863 — 知识增强的检索增强推荐，把外部知识拼进 RAG 流程。

8. **ReGeS（检索-生成互促）** https://arxiv.org/abs/2509.21371 — 对话推荐里让"检索"和"生成"互相喂对方、循环增强。

9. **ERRA（个性化评论检索+aspect）** https://arxiv.org/abs/2306.12657 — 生成解释时先检索用户历史评论、再挑用户最在意的几个 aspect（如"啤酒/氛围"），让解释更具体不空洞。

10. **CIRR（因果不变检索增强）** https://arxiv.org/abs/2512.18683 — 让检索到的证据在分布变化下仍稳健（因果视角去伪相关）。

11. **Enhanced Rec with Retrieval-Augmented LLM (ER2ALM)** https://www.researchgate.net/publication/389370642 — 用 RAG 灵活补充辅助信息、捕捉隐式偏好。

12. **RAG 综述（Gao et al.）** https://arxiv.org/abs/2312.10997 — RAG 领域最常被引的综述，建立 naive/advanced/modular RAG 全景。**入门必扫。**

13. **RAG 评估综述** https://arxiv.org/abs/2504.14891 — 系统梳理"怎么评价一个 RAG 系统好不好"。

14. **GraphRAG 综述** https://arxiv.org/abs/2501.00309 — 把 RAG 装上图：图构建→图引导检索→图增强生成。**理解 G-Refer 的上位框架。**

15. **DyG-RAG（动态图 RAG）** https://arxiv.org/abs/2507.13396 — 面向随时间变化的事件图做检索推理。

16. ⭐ **路径感知最小充分子图检索** https://arxiv.org/abs/2603.07179 — 用信息论准则，只挑"最小且够用"的推理路径集，不多不少。**缝到G-Refer**：把"取 k 条最短路"升级成"取信息论最小充分路径集"，k 自适应、解释不冗余。

17. **TagRAG（标签引导分层 KG-RAG）** https://arxiv.org/abs/2601.05254 — 用标签层级组织知识图谱检索。

18. **知识导向 RAG 综述** https://arxiv.org/abs/2503.10677 — 偏"知识"的 RAG 方法梳理。

19. **MARC（多模态多任务 agentic RAG 冷启动）** https://arxiv.org/abs/2511.08181 — 面向冷启动的多模态 agent 检索推荐。

20. **SemRAG（语义知识增强 RAG）** https://arxiv.org/abs/2507.21110 — 语义分块 + 知识图谱增强问答。

21. **CRAG（协同检索增强 LLM 对话推荐）** https://cseweb.ucsd.edu/~jmcauley/pdfs/kdd24a.pdf — 首个把 LLM 和协同过滤揉进对话推荐检索的工作。

22. **RRCM（排序驱动的协同+元记忆检索）** https://arxiv.org/abs/2605.07129 — 给 LLM 推荐配一个"从协同记忆里按排序检索"的模块。

23. **RaSeRec（检索增强序列推荐）** https://arxiv.org/abs/2412.18378 — 序列推荐时检索相似历史序列当额外上下文。

24. **RUEL（浏览日志检索增强用户表示）** https://arxiv.org/abs/2309.10469 — 用浏览器边端日志检索增强用户画像。

25. **ReACT（检索增强+时序+协同）** https://www.sciencedirect.com/science/article/pii/S030645732500295X — 用时序点互信息检索相关历史行为、自适应挑最有信息量的行为。

26. **Retrieval-Oriented Knowledge for CTR** https://arxiv.org/abs/2404.18304 — 把检索到的知识注入 CTR 预估。

27. **GNN-Guided Prompting KG-RAG** https://openreview.net/forum?id=R1NWMExESj — 用 GNN 引导 LLM 的 KG 检索提示。

28. **Semantic Design Space of RAG Rec（综述）** https://www.sciencedirect.com/org/science/article/pii/S1546221826004558 — 系统综述 LLM-based 检索增强推荐的设计空间。**帮你定位 G-Refer 在版图里的位置。**

# 方向二：找 top-N（候选生成 / GNN 协同过滤 / 负采样）
> 大白话：怎么又快又准地从海量物品里挑出最相关的一批。对应 G-Refer 的"取 top-k 节点"。

29. ⭐ **LightGCN** https://arxiv.org/abs/2002.02126 — 图协同过滤的经典基座：只保留"邻居信息聚合"，砍掉花哨的特征变换和非线性，反而更强更快。**缝到G-Refer**：可当更强/更快的 GNN 打分基座替代 R-GCN。

30. **NGCF（神经图协同过滤）** https://arxiv.org/abs/1905.08108 — LightGCN 的前身，首次把 GCN 系统用于 CF（后被证明有冗余，见 LightGCN）。

31. **PinSage / GCN Web 级推荐** https://arxiv.org/abs/1806.01973 — Pinterest 把 GCN 做到十亿级，工业候选生成经典。

32. **Graph Neural ODE 协同过滤** https://arxiv.org/abs/2311.12329 — 用常微分方程视角建模图上信息传播的连续演化。

33. **LFGCF（轻量标签图协同过滤）** https://arxiv.org/abs/2208.03454 — 面向标签(folksonomy)场景的轻量图 CF。

34. **LightGNN** https://dl.acm.org/doi/10.1145/3701551.3703536 — 更简的图神经推荐，进一步剪枝。

35. **Top-K 推荐 1-bit 量化** https://arxiv.org/abs/2112.01944 — 把用户/物品表示压到 1 比特还能保住 top-k 精度，省内存。

36. **GNNCL（对比学习图推荐）** https://link.springer.com/article/10.1007/s11063-024-11545-9 — 图推荐+对比学习。

37. **频率视角的特征蒸馏推荐** https://arxiv.org/abs/2411.10676 — 从频域看推荐里的知识蒸馏。

38. **CHEST（异构子图 Transformer top-N）** https://arxiv.org/abs/2106.06722 — 为每次交互抽相关异构子图、编码路径语义、课程预训练。（见 Agent 版 #10）

39. **两塔检索模型（背景）** https://developers.google.com/machine-learning/recommendation/dnn/retrieval — 工业召回标配：用户塔/物品塔各出向量，ANN 快速召回。

40. **PinnerSage** https://arxiv.org/abs/2007.03634 — Pinterest 多模态多向量用户表示。

41. **ANN-Benchmarks** https://www.researchgate.net/publication/331280060 — 近似最近邻检索算法的评测基准（HNSW/LSH 等）。

42. **Twitter 候选生成的数据偏差** https://arxiv.org/abs/2105.09293 — 工业召回里的偏差问题与应对。

43. **TwERC（Twitter 集成候选生成）** https://arxiv.org/abs/2302.13915 — 高性能广告候选生成，多路集成。

44. **端到端多样化召回** https://arxiv.org/abs/2305.15534 — 在召回/推荐里实用地保多样性。

45. **自适应硬度负采样 CF** https://arxiv.org/abs/2401.05191 — 负样本别总用一样难度，按训练状态自适应挑合适难度的负例。

46. **SCONE（随机采样造对比视图+硬负例）** https://arxiv.org/abs/2405.00287 — 一套随机采样同时生成对比视图和硬负样本。

47. **用 LLM 做硬负采样** https://arxiv.org/abs/2504.04726 — 让大模型帮你挑"看起来像正例其实是负例"的难负样本。

48. **多样性增强负采样** https://arxiv.org/abs/2508.14468 — 负采样时兼顾多样性。

49. **自动化负采样（隐式反馈）** https://arxiv.org/abs/2311.03526 — 自动搜负采样策略。

50. **MixGCF** https://dl.acm.org/doi/10.1145/3447548.3467408 — 用邻居 embedding 合成硬负例，图 CF 常用。

51. **可靠负采样（社区流行度）** https://arxiv.org/abs/2602.18759 — 用社区内流行度选更可靠的负样本。

52. **多模态推荐负样本生成** https://arxiv.org/abs/2501.15183 — 多模态场景造负例。

53. **公平性负采样** https://arxiv.org/abs/2304.07487 — 负采样对不同用户群的公平性。

# 方向三：粗排 / 精排 / 重排 / 级联
> 大白话：多阶段漏斗里"初筛→决赛→排顺序"各阶段怎么设计、怎么串起来。对应 G-Refer 的"多阶段 top-k 筛选"。

54. **COLD** https://arxiv.org/abs/2007.16122 — 粗排奠基作：粗排不必是"廉价替身"，在算力预算内也能上带交叉特征的复杂模型。（详见你另一份《粗排精排》文档）

55. ⭐ **Ranking Consistency（粗排一致性）** https://arxiv.org/abs/2205.01289 — 粗排真正该优化的不是自己准，而是"和精排排序一致，别提前误杀好货"；提出 RCS 指标。**缝到G-Refer**：给"路径检索器逼近黑盒 GNN"一个可量化目标和评估指标。

56. **RankFlow** https://dl.acm.org/doi/10.1145/3477495.3532050 — 级联各阶段联合训练、消除阶段偏差；下游反馈上游。（详见《粗排精排》文档）

57. ⭐ **IntTower** https://arxiv.org/abs/2210.09890 — 双塔粗排里加细粒度交互：Light-SE 特征加权 + FE-Block 早期跨塔交互 + sum-max 相似度（替代点积），又准又不慢。**缝到G-Refer**：把节点检索的点积换成 sum-max，检索更准。

58. **RankTower** https://arxiv.org/abs/2407.12385 — 双塔粗排增强：多头门控 + 门控交叉注意力 + 最大相似层，蒸馏+LTR 混合损失。

59. **可学全交互双塔** https://arxiv.org/abs/2509.12948 — 让双塔也能做"充分交互"又保效率。

60. **HIT（分层交互增强双塔）** https://dl.acm.org/doi/10.1145/3746252.3761501 — 层次化交互的双塔粗排。

61. **CS3（双塔在线能力协同）** https://arxiv.org/abs/2604.22761 — 在线高效的双塔能力协同。

62. **异构感知粗排** https://arxiv.org/abs/2603.03770 — 粗排训练样本来源异构（召回/精排/曝光），要区别对待。

63. **粗排中间态挑战（KDD24）** https://dl.acm.org/doi/10.1145/3637528.3671580 — 系统分析多阶段级联里粗排的"夹心层"难题。

64. **全链路跨域粗排** https://arxiv.org/abs/2310.08039 — 重新思考大规模粗排：整链路跨域建模。

65. **混合跨阶段协调粗排** https://arxiv.org/abs/2502.10284 — 让粗排和相邻阶段协调。

66. **一致且可泛化的电商粗排** https://arxiv.org/abs/2405.05606 — 电商搜索里做排序一致 + 可泛化的粗排。

67. **COPR（一致性导向粗排）** https://arxiv.org/abs/2306.03516 — 在线广告里把"和精排一致"作为粗排优化目标。

68. ⭐ **LCRON（级联作为一个网络）** https://arxiv.org/abs/2503.09492 — 把整条漏斗当一个网络端到端训：定义"真值穿过所有阶段的联合存活概率"作损失，用可微排序(NeuralSort)连接各阶段。**缝到G-Refer**：把"检索器→LLM"当两阶段级联端到端训，LLM 的梯度反传去指导检索选谁。

69. **全阶段 LTR 统一框架** https://arxiv.org/abs/2405.04844 — 先估下游阶段的选择偏差，再让排序模型顺应它。

70. **自适应神经排序框架** https://arxiv.org/abs/2310.10462 — 面向级联的、对齐业务目标的排序框架。

71. **级联排序联合优化** https://www.researchgate.net/publication/331662734 — WSDM'19，级联排序联合优化早期工作。

72. ⭐ **DFTopK（可微快速 Top-K）** https://arxiv.org/abs/2510.11472 — 用 sigmoid 在第 k/k+1 名中点做软阈值，O(n) 可微 top-k，工业 A/B +1.77% 营收。**缝到G-Refer**：把节点检索的硬 top-k 换成可微版，才能被下游 LLM 的 loss 反传（配合 LCRON）。注意：作者说它不适合直接替 Dijkstra 图搜索。

73. ⭐ **GNRR（语料图上图重排）** https://arxiv.org/abs/2406.11720 — 别孤立给候选打分，把候选组成图、用 GNN 让它们互相参照再排序。**缝到G-Refer**：把独立选出的 top-k 节点组成小图跑一轮 GNN 互相校正、去冗余再喂 LLM。低成本高收益。

74. **神经重排综述** https://arxiv.org/abs/2202.06602 — 多阶段推荐里"重排"的系统综述。**理解重排全景。**

75. **GRN（生成式重排网络）** https://arxiv.org/abs/2104.00860 — 上下文感知地生成整个重排列表。

76. **神经重排个性化多样性** https://www.ruizhang.info/publications/ICDE%202023%20Personalized%20Diversification%20for%20Neural%20Re-ranking%20in%20Recommendation.pdf — 重排时按用户口味平衡相关性与多样性。

77. **去噪神经重排** https://arxiv.org/abs/2509.18736 — 重排阶段去噪。

78. **全局动作空间生成式重排** https://arxiv.org/abs/2604.25291 — 用全局 ID 而非局部下标做生成式重排。

79. **个性化列表价值估计重排** https://arxiv.org/abs/2508.02242 — 从"生成到消费"估计整列表价值。

80. **Ranking Distillation** https://jiaxit.github.io/resources/kdd18ranking.pdf — 鼻祖：把老师(大模型)的 top-K 当额外监督，教学生(小模型)也给这些高分。

81. **RankDistil** https://research.google/pubs/rankdistil-distillation-for-ranking/ — 专为 top-k 排序设计的蒸馏：只对齐 top 部分。

82. ⭐ **HetComp（异构教师 easy-to-hard 蒸馏）** https://arxiv.org/abs/2303.01130 — 从多个强模型的"训练轨迹"里造由易到难的排序知识，自适应蒸馏给轻量学生。**缝到G-Refer**：用强 GNN 的 top-k 轨迹课程式蒸馏给可解释检索器，让它逼近黑盒模型。

83. ⭐ **Cooperative Retriever & Ranker** https://arxiv.org/abs/2206.14649 — 检索器和排序器别各练各的，联合训练、处理分布漂移和假负。**缝到G-Refer**：作"检索器 ⇄ LLM 协同训练"的范式参考。

84. **OneRec（统一召回与排序的生成式推荐）** https://arxiv.org/abs/2502.18965 — 用一个生成式模型统一召回+排序，偏好对齐。**前沿方向。**

# 方向四：知识图谱 embedding / 节点融合邻居 / 路径推理 / GNN 可解释
> 大白话：怎么给图里的节点算好向量（关键是"融合邻居信息"），怎么在图上找解释路径，怎么让 GNN 的决策可解释。这是 G-Refer 检索器的技术核心。

85. ⭐ **KGAT** https://arxiv.org/abs/1905.07854 — 在"用户-物品-属性"大图上，用注意力区分邻居重要性、逐层传播，显式建高阶关系。**缝到G-Refer**：把 R-GCN 换成带注意力的邻居聚合，节点嵌入更强。

86. **KGCN** https://arxiv.org/abs/1904.12575 — 在 KG 上采样固定数目邻居来聚合物品表示，经典"节点融合邻居"。

87. **KGIN** https://arxiv.org/abs/2102.07057 — 在聚合层注入关系 embedding，建模用户对不同关系的"意图"。（Agent 版 #11）

88. ⭐ **RippleNet** https://arxiv.org/abs/1803.03467 — 从用户历史点击出发，沿 KG 边像水波一样多跳扩散偏好，自动发现"历史项→候选项"的路径。**缝到G-Refer**：用偏好传播自动生成路径候选，减少对最短路搜索的依赖。

89. **KGCL（KG 对比学习）** https://arxiv.org/abs/2205.00976 — 对 KG 和用户-物品图做边删除增强，再对比学习，抗噪抗稀疏。

90. ⭐ **KGRec（自监督理性化）** https://arxiv.org/abs/2307.02759 — 给每条知识三元组打"它是不是交互背后真正理由"的分，按 head 邻居数全局归一（消度数偏置），掩码重建+对比剪噪声。**缝到G-Refer**：用它的重要性打分替/补 PaGE-Link 边权，全局度数归一天然契合你"惩罚高度数节点"。

91. **HGT（异构图 Transformer）** https://arxiv.org/abs/2003.01332 — 用自注意力做异构消息传递，自动学关系专属模式、无需手工元路径。

92. **MAGNN（元路径聚合）** https://arxiv.org/abs/2002.01680 — 同时做元路径内 + 元路径间聚合，融多种语义。

93. **去偏邻居聚合** https://arxiv.org/abs/2208.08847 — 修正 GNN 推荐里邻居聚合的偏差。

94. **概念感知去噪 GNN（微视频）** https://arxiv.org/abs/2109.13527 — 聚合时按概念去噪。

95. **KHGCN（层次图胶囊网络）** https://pmc.ncbi.nlm.nih.gov/articles/PMC10137578/ — 知识增强 + 层次胶囊网络。

96. **DCGL（双通道图学习+LLM）** https://arxiv.org/abs/2605.07314 — 双通道图 + 大模型做知识感知推荐。

97. **关联规则增强 KGAT** https://arxiv.org/abs/2011.08431 — 把关联规则挖掘融进 KGAT。

98. **PGPR（策略引导路径推理）** https://arxiv.org/abs/1906.05237 — 用强化学习在 KG 上走路径做推荐，走出来的路径就是解释；软奖励 + 动作剪枝。**经典可解释路径推理。**

99. **KRRL（自监督 RL 知识推理）** https://zhouxiuze.com/pub/KRRL.pdf — 多层表示 + 自监督 RL 引导 KG 上的路径推理。

100. **PaGE-Link** https://arxiv.org/abs/2302.12465 — 为异构图链路预测生成"路径式解释"：k-core 剪枝 + 掩码学习 + 最短路取 top 路径。**G-Refer 路径检索器就来自它。**

101. ⭐ **Power-Link（路径式 KG 补全解释）** https://arxiv.org/abs/2401.02290 — 改进 PaGE-Link：Triplet Edge Scorer 打边权 + 用 graph-powering 矩阵幂替代最短路搜索，可并行、避免误差累积。**缝到G-Refer**：直接换掉你的 Dijkstra 瓶颈。**最高优先级之一。**

102. **eXpath（本体闭合路径规则解释）** https://arxiv.org/html/2412.04846v1 — 用本体闭合路径规则解释 KG 链路预测。

103. **Top-k 最短距离链路预测** https://arxiv.org/abs/1705.02936 — 用 top-k 最短距离做链路预测，经典路径特征。

104. **BioPathNet（神经 Bellman-Ford）** https://www.nature.com/articles/s41551-025-01598-z — 基于路径的链路预测，学节点对间沿路径的表示。

105. **社区影响路径解释** https://www.sciencedirect.com/science/article/abs/pii/S0893608025005258 — 异构图链路预测的社区影响路径解释。

106. ⭐ **GraphLoRA（结构感知 LoRA）** https://arxiv.org/abs/2606.07526 — 在 LoRA 低秩通路里嵌图消息传递，让协同拓扑直接参与参数更新。**缝到G-Refer**：把你现在的 vanilla LoRA(rank=8)升级成能吸收图结构的版本。

107. **TokenRec（把 ID 词元化做生成式推荐）** https://wenqifan03.github.io/papers/2024-TokenRec.pdf — 学习把用户/物品 ID 拆成词元，供 LLM 生成式推荐。

108. ⭐ **FACE（CF embedding→LLM token）** https://arxiv.org/abs/2510.15729 — 向量量化+codebook 把连续 CF 向量离散成 token，和文本一起喂 LLM。**缝到G-Refer**：把检索到的节点 CF 向量量化成 token 注入 LLM（soft-prompt 的替代路线）。

109. **LLM 是否受益于用户/物品 embedding** https://www.arxiv.org/pdf/2601.04690 — 系统研究把 CF embedding 注入 LLM 到底有没有用。

110. **LLMRec（图增强 LLM 推荐）** https://www.atailab.cn/seminar2024Fall/pdf/2024_WSDM_LLMRec... — 用 LLM 做图数据增强（补边/补属性）再推荐。

111. **SO𝒢_k（一个 token 表达图结构）** https://arxiv.org/abs/2602.01771 — 把一整块图拓扑压成一个高选择性 token 让 LLM 理解结构。

112. **图基础模型推荐综述** https://arxiv.org/abs/2502.08346 — 图基础模型用于推荐的全面综述。**建立方向四全景。**

113. **ML-KGCL（多层 KG 对比学习）** https://link.springer.com/chapter/10.1007/978-3-031-30672-3_17 — 多层次的知识图谱对比学习推荐。

114. **LLM 遇上 CF（高效框架）** https://arxiv.org/abs/2404.11343 — 高效地把 LLM 和协同过滤结合。

115. **多层图注意力对比学习知识感知推荐** https://arxiv.org/abs/2605.08499 — 多层 GAT + 对比学习。

116. **KG 增强 + 动态关系感知 + 双视图对比** https://www.sciencedirect.com/science/article/pii/S0957417425042514 — 上下文感知动态关系 + 双视图层次对比。

117. **SGL（图自监督学习）** https://arxiv.org/abs/2010.10783 — 首个把对比自监督用于推荐，节点/边 dropout 造对比视图。**KGCL 等的源头。**
