# G-Refer

可解释性推荐：指出推荐结果背后的原因



# 前置知识

协同过滤CF：根据用户-商品的历史交互记录（购买、点击等行为），推断用户偏好的一类方法。其底层假设是"行为相似的用户，偏好也相似"。

在这篇论文里，CF 信息分两种粒度：**结构性 CF**（谁和谁产生过交互）和**语义性 CF**（谁的属性内容和谁相似）



# 研究方法

## 混合图检索

推荐一个物品可以从结构和语义上解释为什么

结构：路径 u1 → i1 → u2 → i2，代表"u1 和 u2 都交互过 i1，u2 又交互了 i2，因此推断 u1 也可能喜欢 i2"

语义：i2 和 i3 可能因为主演都是一个人，这种内容层面的关联而彼此相似



节点/路径/子图 在检索的时候都有优势，论文采用了混合检索（节点+路径）来获取CF

获取的节点+路径会被转化成通俗的文本为后续生成推荐理由提供支持



### 路径检索器

路径检索的底层任务是**链路预测**——即给定用户 u，预测其可能与哪个商品 i 之间应该存在一条边，这是一个纯粹的图算法问题。

GNN 虽然能很好地建模图结构，捕捉 CF 信息，但本身缺乏可解释性。作者设计了一个路径级检索器，为已经训练好的 GNN 预测结果生成事后解释，把 GNN 的结果还原成显式的路径。

1. 用一个L跳的GNN（R-GCN），为每个用户生成一个向量，为每个商品生成一个向量，内积越大说明越相关
2. m-core剪枝，策略：把度数高的剪掉，因为热门物品的关系没有什么参考价值（大家都买过这个物品，说了等于没说）
3. PaGE-Link计算每条边在推荐算法中的权重，得到0-1的值，方法是做个掩码看对结果影响大不大
4. 损失函数=预测损失+路径损失

**预测损失**是基于扰动式解释（我猜这个东西重要，我就把它mask住，如果变了就说明真的重要）

**路径损失**是挑选出能构成路径的边，给每一个路径打分，挑出更短&不包含高度数节点的边

举例：两条路径（A-用户B-I，A-明星C-I）都能连我A到商品I，会选择用户B这条边，因为度数小



**收敛之后**，再跑一次 Dijkstra，就能得到最多 k 条解释路径，作为路径级 CF 证据。



### 节点检索器

因为图是有噪声&不完整的，仅靠路径是不够的

用SentenceBERT把用户信息和商品信息编码成向量，做相似度检索

挑选出**top-K个**最相关的用户节点和**top-K个**最相关的商品节点



### 图翻译

检索后，将K个路径，K个用户节点，K个商品节点，喂给LLM做解释

```
prompt:
结合商品标题、商品简介和用户简介，请说明该用户会喜欢此商品的原因。商品标题：[商品标题]。商品简介：[商品概述]。用户简介：[用户概述]。针对该用户-商品组合，以下是一些相关用户和商品。用户：[前𝑘个相似用户简介]。商品：[前𝑘个相似商品简介]。针对该用户-商品组合，以下是几条通过用户与商品的互动行为连接用户和商品的相关路径。[前𝑘条解释路径]（每条格式为“<用户简介> -> 购买 -> <商品简介> -> 被……购买 -> <用户简介> -> ……”）。解释：
```



### 检索粒度选择

检索粒度不选择子图：搜索空间小于子图，子图包含高阶复杂信息

【我的创新点】可以套用K-ragrec的方法，尝试做子图级别的检索



## 知识剪枝

对于一些用户-商品仅凭简介就能生成介绍了，使用多余的信息反而会削弱LLM使用CF信息

删除一些过于简单（只用简介即可生成的答案）的数据集

剪枝策略：算简介文本vs人类写的标准答案解释之间的相似度

Bert(用户简介 拼接 商品简介) ==余弦相似度== Bert(标准答案解释)

引入剪枝比例t，高于t的会被留下来（70%）

相似度高的会被砍掉，删除简单样本，保留基于 CF 的解释才能解释清楚的样本



## 检索增强微调

微调的LLM能更好的利用检索到的CF信息生成解释

让LLM忽略错误的CF信息，依靠自身内部知识进行生成

Lora

输入：用户-商品对，用户简介，商品简介、与之相关的检索知识，以及一个对应的提问 Q

输出：文本解释 E

损失：-log(生成标准解释)





# 实验

## 实验方案

传统指标不行

GPTscore：GPT-3.5-turb，LLM评估LLM

BERT(Recall)score：对比："啤酒、氛围、便宜"，匹配到了就高分

BERT(Precision)score：对比："啤酒、音乐、氛围、干净"，匹配到了就高分

BARTscore：使用BART模型，判断生成解释转到标准答案是否容易

BLEURT：专门的模型判断两个句子像不像「训好的模型还是我们要自己训？」

USR：看所有句子统计有多少条是完全独一无二的



BERT把解释所有词拆分，生成向量

例：标准答案:"啤酒、氛围、便宜"，生成解释:"啤酒、音乐、氛围、干净"

BART：把******生成的解释******喂给模型当"已知信息",然后让模型去"生成标准答案",统计这个过程中每一步生成对词的概率有多高,累乘/累加起来当作最终分数。



对比模型：NRT [35 ], Att2Seq [16 ], PETER [ 33 ], PEPLER [ 34 ], and XRec [ 50 ]



### 实验参数

节点嵌入维度设为128

最大检索路径长度设为5

m取值为2

SentenceBERT作为编码器

检索路径的数量以及检索节点（包括用户和物品）的数量均设为2

剪枝比例t均设为70%

LLaMA 2-7B & LLaMA 3-8B

8*A100

学习率、轮数和最大长度分别设置为2e-5、2和2048

LoRA适配器的秩设置为8

温度设为0

最大输出标记数设为256

GPT-3.5-turbo模型计算GPTScore指标

arXiv:2604.03666

把 G-Refer 扩到多模态——RQ-VAE 融合图文 + 规则启发式路径检索替代可学习边权（快 3–6倍更稳），BERTScore-F1 比 G-Refer 高 0.56%–4.01%。



## Retrieval-Augmented Explanation with Hierarchical Aggregation

arXiv:2507.09188

直击 G-Refer 两大痛点：分层聚合多级摘要取代随机评论采样（治 Profile Deviation）

向量检索取代 O(N²) Dijkstra（推理 4 分钟→<1 秒），BERTScore-Precision 高 7.46%–26.8%。
